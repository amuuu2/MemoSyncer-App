\# 📦 MemoSyncer Mobile — 业务需求文档（精简落地版）



\## 1. 产品定位与核心价值



MemoSyncer Mobile 是一款\*\*基于 AI 驱动、离线优先的渐进式双语闪卡（Flashcard）复习 App\*\*。

用户通过\*\*手动粘贴\*\*长文本、笔记或阅读材料，利用大模型将其解构为原子化的双语记忆卡片，并由标准 SM-2 算法驱动日常复习，帮助用户利用碎片化时间建立长期的跨语言知识记忆。



\---



\## 2. 核心用户场景（User Stories）



\* \*\*场景 A（快速输入生成）：\*\* 用户在其他 App 看到一段干货或背诵材料，复制后打开 MemoSyncer，直接粘贴到输入框中。AI 瞬间将其切碎，转化为一组中英双语的闪卡。

\* \*\*场景 B（离线地铁复习）：\*\* 用户在早高峰地铁上（无网络信号），打开 App 进入“今日复习”。得益于纯本地数据库，用户依然可以流畅地进行 3D 卡片翻转背诵并评分。

\* \*\*场景 C（双语进阶）：\*\* 用户在复习专业术语时，可以一键切换中/英双语表述，在记忆知识点的同时熟悉对应的英文技术表达。

\## 4. 功能模块拆解



\### 4.1 知识输入与 AI 解构模块



\* \*\*界面交互：\*\* 一个极简的文本输入框（Textarea），支持一键“粘贴”和“清除”。

\* \*\*AI 动态切碎：\*\* 大模型接收用户输入的纯文本，将其提炼出 5-15 张最核心的卡片。每张卡片严格包含：中文问题、英文问题、中文答案、英文答案以及细分知识标签。



\### 4.2 沉浸式复习与算法引擎模块



\* \*\*界面交互：\*\* 采用纯黑主题。卡片支持点击 3D 翻转（正面看问题、背面看答案）。底部提供三个记忆反馈按钮（1分：彻底遗忘；3分：勉强想起；5分：完美记住）。

\* \*\*算法逻辑：\*\* 根据用户评分，算法在本地动态调整卡片的 `repetitions`（连续正确次数）和 `easiness\_factor`（难度系数），计算出纯日期格式的 `next\_review\_date`。



\### 4.3 记忆可视化（数据看板）



\* \*\*本地热力图：\*\* 提供 365 天复习热力图，颜色深浅代表当天复习的卡片总次数。

\* \*\*基础统计：\*\* 显示“今日待复习卡片数”、“已掌握卡片总数”、“连续打卡天数”。

\## 5. 核心数据字典



\### 5.1 卡片集表 (Deck)



| 字段名 | 类型 | 说明 |

| --- | --- | --- |

| id | int / uuid (PK) | 唯一标识符 |

| title | string | AI 根据文本主题自动生成的卡片集名称 |

| created\_at | datetime | 创建时间 |



\### 5.2 闪卡明细表 (Flashcard)



| 字段名 | 类型 | 说明 |

| --- | --- | --- |

| id | int / uuid (PK) | 唯一标识符 |

| deck\_id | int / uuid (FK) | 关联的卡片集 ID |

| question\_zh / en | string | 中/英文核心问题或概念 |

| answer\_zh / en | string | 中/英文精炼解答 |

| knowledge\_tag | string | 细分知识点标签（如：#计算机网络） |



\### 5.3 算法复习状态表 (ReviewState)



| 字段名 | 类型 | 默认值 | 说明 |

| --- | --- | --- | --- |

| card\_id | int / uuid (PK, FK) | - | 关联的闪卡 ID |

| easiness\_factor | double | 2.5 | 记忆难度系数（标准 SM-2 起始值） |

| interval | int | 0 | 下次复习的间隔天数 |

| repetitions | int | 0 | 连续复习正确的次数 |

| next\_review | date | Today | 下一次需要复习的绝对日期 |
